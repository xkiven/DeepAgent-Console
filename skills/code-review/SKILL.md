# Code Review

当用户要求进行代码评审、缺陷排查、回归评估或风险分析时，使用这个技能。

工作流程：
1. 在评价代码变更前，先阅读目标文件和必要上下文。
2. 重点关注正确性、用户可见回归、安全问题和测试覆盖缺口。
3. 优先给出带文件依据的具体发现，而不是泛泛的风格意见。
4. 如果没有发现缺陷级问题，要明确说明，并补充剩余风险。
